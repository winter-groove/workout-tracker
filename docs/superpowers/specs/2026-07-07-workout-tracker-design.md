# 운동 기록 앱 (Workout Tracker) 설계 문서

날짜: 2026-07-07
상태: 사용자 승인 완료

## 개요

헬스장에서 점진적 과부하(progressive overload)를 실천하기 위한 개인용 운동 기록 PWA.
운동별로 세트마다 무게×횟수를 기록하고, 운동을 시작할 때 지난번 기록을 바로 보여줘서
"지난번보다 조금 더"를 쉽게 만든다. 플랜핏(Planfit)의 기록 UX를 참고한 클론 성격.

## 확정된 요구사항

- 플랫폼: 모바일 웹앱(PWA). 홈 화면 추가로 앱처럼 사용, 오프라인 동작.
- 기록 단위: 운동별 세트마다 무게(kg) × 횟수(reps). 세트별 완료 체크.
- 점진적 과부하 지원: 운동 시작 시 해당 운동의 지난번 기록(무게×횟수 목록)을 표시하고,
  새 세트 입력칸에 지난 기록 값을 미리 채운다.
- 운동 목록: 내장 운동 라이브러리(이미지 포함) + 직접 등록(아이콘 선택).
- 루틴 템플릿: "가슴 날" 같은 운동 묶음을 만들어두고 세션 시작 시 불러오기.
- 데이터 저장: 폰 로컬(IndexedDB)만 사용. 서버/계정 없음. JSON 내보내기/가져오기로 백업.
- 플랜핏 UX 반영: 세트 완료 체크 시 자동 휴식 타이머, 세션 진행률/운동 시간 표시,
  주간 운동 체크 표시, 운동마다 실제 동작 사진.

## 범위 제외 (이번 버전에서 하지 않는 것)

- AI 루틴 추천, 음성 코칭, 커뮤니티 (서버/AI 필요 — 플랜핏에는 있으나 제외)
- 진전 그래프/차트 (다음 버전 후보)
- 목표 자동 제안 (증량 로직)
- 서버 동기화, 계정, 멀티 디바이스
- 운동 자세 설명 일러스트/영상 라이브러리
- 사진 직접 업로드 (직접 등록 운동은 픽토그램 아이콘 선택)

## 기술 스택

- React + Vite (TypeScript)
- vite-plugin-pwa: 서비스 워커, 오프라인 캐시, 홈 화면 설치
- Dexie.js: IndexedDB 래퍼
- 배포: 정적 호스팅 (GitHub Pages 또는 Netlify)
- 이미지: free-exercise-db (퍼블릭 도메인)에서 선별한 운동 사진을 앱에 번들
  (오프라인에서도 보여야 하므로 원격 로드 금지)

## 화면 구성

하단 탭 3개: 홈 / 기록 / 관리. 룩앤필은 승인된 목업 기준
(흰 배경, 굵은 타이포, 파란 포인트 #3182F6, 둥근 카드, 다크 네이비 휴식 타이머 바).
목업 파일: `.superpowers/brainstorm/*/content/planfit-style-v2.html`

### 홈

- 다음 루틴 카드(마지막으로 안 한 루틴 또는 최근 루틴)와 "운동 시작하기" 버튼
- 루틴 선택 또는 빈 세션으로 시작 가능
- 이번 주 요일별 운동 완료 체크 표시
- 최근 세션 목록 (날짜, 루틴명, 운동 개수)

### 운동 세션 (핵심 화면)

- 운동 하나가 화면 전체를 차지. 이전/다음 운동 버튼으로 이동, 상단에 진행률(2/5)과 경과 시간
- 운동 사진 + 이름 + 부위/기구 태그
- 지난번 기록 배지: "지난번 60kg×10 · 60×8 · 55×10"
- 세트 목록: 세트번호 / 무게 / 횟수 / 완료 체크. 지난 기록 값이 미리 채워짐.
  무게·횟수는 탭해서 수정 (숫자 입력)
- 세트 완료 체크 → 자동 휴식 타이머 시작 (기본 90초, 설정 가능, 건너뛰기 버튼)
- 세트 추가 / 운동 추가 / 세션 완료
- 세트 체크마다 즉시 저장 → 앱이 꺼져도 진행 중 세션 복구

### 기록

- 세션 목록을 날짜 역순으로 표시 (날짜, 루틴명, 운동별 세트 요약)
- 운동별 필터: 특정 운동만 골라 날짜순 기록 확인 (무게 변화 추적)
- 세션 상세 보기 및 삭제

### 관리

- 내 운동 목록: 내장 라이브러리 검색(이름)/필터(부위) + 직접 등록(이름, 부위, 아이콘 선택)
- 루틴 템플릿: 생성/수정/삭제. 운동 순서와 기본 세트 수 지정
- 데이터 내보내기(JSON 다운로드) / 가져오기(JSON 업로드, 전체 교체)
- 휴식 타이머 기본 시간 설정

## 데이터 모델 (Dexie/IndexedDB)

```ts
// exercises — 운동 사전
{
  id: string,            // 내장: 'lib-<slug>', 커스텀: uuid
  name: string,          // 한국어 이름
  bodyPart: string,      // 가슴 | 등 | 하체 | 어깨 | 팔 | 코어 | 기타
  equipment: string,     // 바벨 | 덤벨 | 머신 | 케이블 | 맨몸 | 기타
  imagePath?: string,    // 번들된 이미지 경로 (내장 운동)
  iconKey?: string,      // 픽토그램 키 (커스텀 운동)
  isCustom: boolean,
  isHidden: boolean      // 내장 운동 숨기기용
}

// routines — 루틴 템플릿
{
  id: string,
  name: string,                            // "가슴 날"
  items: { exerciseId: string, defaultSets: number }[]  // 순서 보존
}

// sessions — 운동 기록
{
  id: string,
  startedAt: number,     // epoch ms
  finishedAt?: number,   // 미완료(진행 중)면 없음
  routineName?: string,
  entries: {
    exerciseId: string,
    sets: { weight: number, reps: number, completedAt?: number }[]
  }[]
}
```

- 인덱스: `sessions.startedAt` (최근순 조회), 지난 기록 조회는 최근 세션들을
  역순으로 스캔해 해당 exerciseId의 첫 매치를 사용
- 진행 중 세션: `finishedAt`이 없는 세션이 있으면 홈에서 "이어서 하기" 제안
- 내장 운동 시딩: 첫 실행 시 라이브러리 JSON을 exercises 테이블에 삽입,
  버전 키로 관리해 앱 업데이트 시 신규 운동만 추가

## 내장 운동 라이브러리

- free-exercise-db에서 헬스장 주요 운동 약 100개 선별
- 각 운동에 한국어 이름 매핑 (예: Barbell Bench Press → 벤치프레스)
- 이미지는 첫 프레임(0.jpg) 1장씩 앱 assets에 포함, 빌드 시 최적화(리사이즈/webp)
- 라이브러리 데이터는 JSON 파일로 관리 (id, 한국어명, 부위, 기구, 이미지 경로)

## 오류 처리

- IndexedDB 쓰기 실패: 사용자에게 토스트로 알리고 해당 입력 유지 (재시도 가능)
- 진행 중 세션 복구: 앱 재실행 시 미완료 세션 감지 → 이어하기/버리기 선택
- JSON 가져오기: 스키마 검증 후 전체 교체, 실패 시 기존 데이터 보존
- 저장 공간: IndexedDB 용량 초과는 사실상 발생하지 않는 규모(텍스트 데이터)라 별도 처리 없음

## 테스트

- 데이터 계층 단위 테스트 (Vitest + fake-indexeddb):
  세션 저장/조회, 지난 기록 조회, 루틴 CRUD, 내보내기/가져오기 왕복
- 컴포넌트 테스트는 핵심 플로우(세트 체크 → 저장) 위주로 최소한만
- 수동 검증: 폰 브라우저에서 PWA 설치, 오프라인 동작, 세션 복구

## 성공 기준

- 헬스장에서 폰으로: 루틴 선택 → 운동별로 지난 기록 보며 세트 기록 → 완료까지
  막힘 없이 가능
- 비행기 모드에서도 전체 기능 동작
- 지난번 기록이 세트 입력칸에 미리 채워져, 무게를 올릴지 그대로 갈지만 결정하면 됨
